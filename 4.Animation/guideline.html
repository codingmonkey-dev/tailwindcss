요소 변화: 200-300ms - 페이지 전환: 300-500ms - 복잡한 애니메이션: 500ms+ 성능 고려사항 ------------ - transform과 opacity는 GPU 가속 사용 (성능 최적화) - width, height, margin, padding 등은 레이아웃 재계산이 필요 (성능 저하) - will-change 속성 고려 (필요한 경우만) 접근성 고려사항 -------------- - 사용자가 애니메이션을 선호하지 않는 경우 고려 - 너무 빠르거나 강한 애니메이션은 피하기 - 중요한 정보는 애니메이션에만 의존하지 않기 📱 반응형 애니메이션 ================== 반응형 브레이크포인트 ------------------ sm: 640px 이상 md: 768px 이상 lg: 1024px 이상 xl: 1280px 이상 2xl: 1536px 이상 반응형 예시 ---------- hover:scale-105 md:hover:scale-110 = 기본: 5% 확대, 768px 이상: 10% 확대 transition-none md:transition-all = 기본: 트랜지션 없음, 768px 이상: 모든 속성 트랜지션 duration-200 lg:duration-300 = 기본: 200ms, 1024px 이상: 300ms 💡 애니메이션 디자인 원칙 ======================= 1. 목적성 - 모든 애니메이션은 명확한 목적이 있어야 함 - 사용자 경험 향상을 위한 수단 2. 일관성 - 비슷한 요소는 비슷한 애니메이션 사용 - Duration과 Easing 일관성 유지 3. 자연스러움 - 현실 세계의 물리법칙 고려 - ease-out이 일반적으로 자연스러움 4. 성능 - GPU 가속 속성 우선 사용 (transform, opacity) - 불필요한 애니메이션 제거 5. 접근성 - 과도한 움직임 피하기 - prefers-reduced-motion 고려 상태별 애니메이션 ---------------- hover: 마우스 호버 focus: 포커스 상태 active: 클릭/터치 상태 group-hover: 부모 요소 호버 group-focus: 부모 요소 포커스 애니메이션 조합 팁 ----------------- 1. 하나의 요소에 여러 transform 조합 가능 hover:scale-105 hover:rotate-3 hover:-translate-y-1 2. 다른 속성과 함께 사용 transition-all duration-300 hover:bg-blue-600 hover:scale-105 hover:shadow-lg 3. 그룹 애니메이션 group-hover:scale-110 group-hover:rotate-6 4. 순차적 애니메이션 delay-100, delay-200, delay-300으로 순차 실행
